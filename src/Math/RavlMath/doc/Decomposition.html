Eigen decomposition or SVD?

Eigen decomposition decomposes a matrix M thus: 

   M * E = E * L
where L is a diagonal matrix of eigenvalues, and E is a matrix of eigenvectors. On the other hand, SVD decomposes M thus: 
   M * V = U * D
where D is a diagonal matrix of singular values, and U and V are orthogonal matrices. In general, D and L are not the same. But it is easy to show that, for symmetrical matrices, these two decompositions are identical: D = L and U = V = E. In this case, you have the choice of which to use. It is generally reckoned that SVD is more stable, and is likely to give a more accurate answer than eigen decomposition.

Which eigen decomposition?

If you are nevertheless using eigen-decomposition, there are currently two algorithms in RAVL: one from the CCMath library and one from the JAMA library. The CCMath ones are the functions named FastEigen..... This algorithm is faster, but occasionally fails for ill-conditioned matrices. The remaining functions are based on the JAMA algorithm implemented in EigenValueC. Note that EigenValueC is templated, so that different floating point representations may be used.

Which SVD?

Similarly there are two algorithms for SVD: the one in SVDC which is a C++ implementation of the JAMA, and the ones in the SVD functions below, which are based on the CCMath implementation.

Intel MKL library

If you have the Intel MKL library and have included the code in Contrib/IntelMKL in your build, it will link in the faster Intel MKL library version in place of the CCMath one. This has implementations of eigen-decomposition and SVD, as well as matrix inversion and FFT.
