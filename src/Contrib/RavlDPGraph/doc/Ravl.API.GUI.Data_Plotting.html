Requires library RavlDPGraph

This library allows the IO mechanism to 'save' data to the display as a graph. The string "@GRAPH" is used in place of a file name.

To save individual images to the display, see this example.

It can also be used to display a sequence of graphs via the RAVL streaming mechanism. Thus if you want to output a series of graphs to the display, the constructor might be invoked as:

  DPOPortC > outp;
  if(!OpenOSequence(outp,"@GRAPH")) {
    cerr << "Failed to open output '" << ofn << "'\n";
    return 1;
  }

To give the window a label "Graphs", invoke it like this:

  if(!OpenOSequence(outp,"@X:Graphs")) {

Normally repeated saves to the same window will replace its contents, if you want to accumulate the results of processing in a window you can use @GRAPHA.
