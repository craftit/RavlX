Some points to note:

  • The transform used by the warper actually computes points in the reference image by transforming pixel coordinates from the warped image. Hence if the warper is constructed on a transform with for example a scale of 2 and positive rotation, the warped image will be ½ of the size of the reference image and rotated negatively with respect to it.

  • There are two possible coordinate systems to use when warping an image:

    1. The coordinate system is at the top left-hand corner of pixel (0,0). Thus a rotation of +90° will map pixel (0,0) to pixel (-1,0).
    2. The coordinate system is at the centre of pixel (0,0), so that a rotation of +90° will map pixel (0,0) to itself.

    The default behaviour is the former, to preserve backwards compatibility. Use the SetMidPixelCorrection() method with correction = false to get the latter behaviour.

  • The default interpolation used is bilinear. This can be changed using suitable definitions of MixerT and SampleT in constructor and template of the relevant warper.
