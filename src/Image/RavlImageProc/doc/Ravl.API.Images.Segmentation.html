The classes on this page are an odd assortment of segmentation methods, written by different authors, with little attempt at a uniform approach. Our apologies.

We can view image segmentation as a 2-stage process:

  1. Identify sets of pixels that are "similar". This could be achieved by simply applying a binary threshold to a grey-level image, or it could be the result of a more sophisticated method based on one or more criteria such as brightness, colour, motion, proximity, ...
  2. Group the pixels into connected regions, using ConnectedComponentsC say, and label each region uniquely, creating a segmentation map.

Depending on the algorithm, some classes (e.g. ChromaThresholdRGBC) just implement the first step, while others such as region growing (RegionGrowC, RegionGrowStealC) naturally perform both steps together.
