This page describes the two basic ways extra documentation can be added to the automatically generated class pages.

Adding extra documentation to the branch nodes

The "branch nodes" are those whose URL (like this page) end as: .../Tree/xxx.html. These are the documentation pages that exist in both User and Develop form. You may add additional information to the tree structure by:

  • Create a normal HTML file (suffix .html) containing your additional documentation. To the <head> section of the HTML document add author and docentry lines: 

        <!--! author="Robert Crida" -->
    <!--! docentry="Ravl.Documentation.Branches" -->

    The docentry entry determines where the page goes in the documentation tree.

  • Add an EHT entry to the defs.mk file.

For documentation added in this way, CxxDoc will add: some standard links; links to the parents of the node, and a heading that is duplicated from the title in the HTML head. If an automatically-generated branch node page already exists at the same URL, its contents will be merged with the new documentation.

Stand-alone pages

If you have information that does not obviously fit in the class documentation tree, or is not HTML, it can be included as a stand-alone item. These files will be automatically copied to the html sub-directory ($PROECT_OUT/share/doc/$PROJECT/html). Despite the name of the directory, documentation added in this manner may be of any format (it can be .html, .pdf, .png or .txt, etc.). Unlike documentation produced using the above "EHT" option, no processing is performed on the source file (even if the source is a HTML document).

To specify documentation to be included in this manner, it should be included as an HTML item in the defs.mk file. No links to the added documentation are automatically created by the CxxDoc documentation system, however, you may manually specify links from your other documentation source; just use the ../../../html/xxx.ext path (where xxx.ext is the name of your extra documentation file).
