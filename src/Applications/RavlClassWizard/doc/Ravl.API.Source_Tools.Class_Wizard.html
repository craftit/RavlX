Usage: cwiz [options] 
   -p () [] Prefix to add to modified files, can be used to check results before applying the changes directly.  
   -v (false) [false] Verbose mode. 
   arg (.) [.] Input directory.  
   -d (false) [false] Dry run, don't write any files. 
   -s (false) [false] Write stubs. 
   -dt (false) [false] Dump parse tree. 
   -help (true) [false] Print usage information.

Class wizard is program designed to help with writing and maintaining Ravl style 'big objects'. You write the body class, exactly as it should eventually be: 

  • #include "Ravl/RefCounter.hh"
  • with class name ending ...BodyC
  • with an entry under HEADERS in the defs.mk file
  • with public inheritance of RCBodyC

If you run cwiz (with no options) it will add the handle class to all appropriately-formatted body classes that it finds, in that directory and any nested directories.

Download this example and try it yourself.

Notes:

  • By default, the header file is overwritten. Use the "-p" option to specify a different o/p file. Use "-p -" to write o/p to the terminal.
  • Uses the public part of the body class as reference to maintain the handle class.
  • Modifies existing header files, no need for seperate header compilation stage.
  • Updates existing handle classes when additions are made to body classes.
  • Works around any modifications made by the author.
  • All methods and classes that are added are marked with '//!author:cwiz'.

Operations:

  • Create handle a method for each public body method.
  • Create handle constructors for each public constructor in the body class.
  • If no handle class is found, creates the handle class and places it after the body class.
  • Remove handle methods and constructors for which there are no body methods. (Only methods marked with '//!author:cwiz')
  • Optionally will create stub functions for the .cc file with '-s' option.
