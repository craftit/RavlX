Requires library RavlDPDisplay3d

Displaying 3D objects and sequences using the "@GL" virtual file

This library allows the IO mechanism to 'save' 3D data to the display. The string "@GL" is used in place of a file name:

  Save("@GL", cube);

"@GL" is one of the virtual files. For a complete example, see here. Use the middle mouse button and the mouse wheel to move and rescale the image in the window.

It can also be used to display a sequence of 3D objects via the RAVL streaming mechanism. Thus if you want to output a series of images to the display, the constructor might be invoked as:

  DPOPortC outp;
  if(!OpenOSequence(outp,"@GL")) {
    cerr << "Failed to open output '" << ofn << "'\n";
    return 1;
  }

To give the window a label "Cube", invoke it like this:

  Save("@GL:Cube", cube);

If you would like to display several images in separate windows, use different labels: @GL:Cube1, @X:Cube2 etc.

Adding 3D objects to images using the "@GLA" virtual file name

Normally repeated saves to the same window will replace its contents. However if you want to accumulate the results of processing in a window you can use "@GLA". For example if you want to add some points:

  Save("@GL:Cube", cube);
  Save("@GLA:Cube", mesh);

This is relevant if you want for example to add graphics to an image.
