Sometimes you want to make code thread-safe because it might> be used in multithreaded code. But you don't want to use the mechanisms in OS->Threads because your code is basically O/S-independent, & so you don't want to pull in the OS library at this stage. If that is the case, use these classes / functions instead. If threading is subsequently used, these functions will be transparently enabled when the RAVL threads library RavlThreads is linked into a program.

Id 0 is a general system lock.
Id 1 is used for locking the random number generator.
Id 2 is used for non-reentrant 'C' library calls.
Id 3 is reserved for use in RavlLogic.
