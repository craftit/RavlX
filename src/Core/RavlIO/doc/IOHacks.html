You want to use the Load() / Save() mechanism (or corresponding streaming system) but it complains at run time? The problem is that QMake doesn't know which library to link, as the decision about which actual function to use is not made until run time. So you have to include the relevant library explicitly in defs.mk. But you don't know the library name - so what to do? Sometimes the documentation will tell you, particularly if you are using image or video classes and files.

But for bog standard classes, there is often no help. The library you need is in $RAVL_INSTALL/lib/RAVL/linux/shared One way to identify it is as follows:

  1. Run QLibs to get a partial list of libraries needed.
  2. Identify the name of the library that includes the class of interest.
  3. There is likely to be a library at the above path of the same name, but with "IO" added, that will fix your problem. E.g. if QLibs indicates that the relevant library is RavlMath, Load() / Save() will need RavlMathIO.
  4. Add this library to your PROGLIBS entry in defs.mk.
